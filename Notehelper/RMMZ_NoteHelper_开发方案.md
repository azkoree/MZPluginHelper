# RMMZ 备注标签助手 — 开发方案

## 一、核心定位

一款离线可用的本地 HTML 应用，用于替代 RMMZ 编辑器内置插件帮助面板的不便体验。用户将插件文件导入后，工具自动从插件帮助文档中提取备注标签（`<...>` 格式），在独立的长文本编辑器中通过点击标签快速组装备注内容，最终复制到 RMMZ 数据库对象的备注栏中。

**关键词：** 标签提取、点击插入、批量管理、配置持久化。

---

## 二、基础功能

| 功能 | 说明 |
|---|---|
| **插件导入** | 点击「导入插件」按钮弹出文件选择（支持多选 `.js`），或直接将文件/文件夹拖拽到窗口完成导入 |
| **导入策略选择** | 如果已有已导入的标签，再次导入时弹出对话框询问：**覆盖**（清空后用新标签替换）/ **追加**（保留当前标签并叠加新标签）/ **取消** |
| **标签自动提取** | 解析插件文件的 `/*: @help ... */` 块，扫描所有形如 `<TagName ...>` 的行，去重后归类展示 |
| **手动添加标签** | 用户可手动输入自定义标签模板，补充插件帮助中未列出的标签 |
| **标签展示 & 点击插入** | 提取到的标签以可点击的按钮形式展示，点击后插入到输入框的当前光标位置 |
| **长文本输入框** | 一个宽敞的可编辑区域，支持光标定位，所有标签插入操作均插入到光标所在位置 |
| **复制输出** | 一键将当前输入框内容复制到系统剪贴板 |
| **导出标签配置** | 将当前插件的标签集合导出为 JSON 文件，下次可直接导入复用 |
| **导入标签配置** | 从之前导出的 JSON 文件中恢复标签配置 |

## 三、技术方案

### 3.1 技术栈

```
NoteHelper/
├── index.html          # 主页面（含完整 UI）
├── style.css           # 样式
├── script.js           # 全部应用逻辑
└── README.md           # 使用说明
```

- **纯前端**：HTML + CSS + Vanilla JS，零依赖，不需要构建工具
- **文件读取**：使用浏览器 `File API` 读取本地 `.js` 文件内容
- **剪贴板写入**：使用 `navigator.clipboard.writeText()`
- **持久化与导出**：`localStorage` 缓存最近会话，导出为 JSON Blob 下载
- **拖拽支持**：通过 `drag` / `drop` 事件处理文件拖入

### 3.2 为什么不需要后端

- 所有操作都在本地浏览器中完成
- 不需要服务器、不需要安装运行时、双击 `index.html` 就能用
- 隐私安全：插件代码和标签数据不会离开本地

## 四、可行性分析

| 维度 | 评估 |
|---|---|
| **标签提取准确性** | RMMZ 插件遵循 `/*: ... */` 注释规范，`@help` 段内用 `*` 前缀的文本行描述标签，提取 `<...>` 模式是成熟的正则匹配，误报率低 |
| **批量导入性能** | 标签提取只扫描注释块而非整个 JS 语法树，147 个插件批量导入预估在毫秒级完成 |
| **跨浏览器兼容** | 使用的 API（File、Clipboard、localStorage、Drag & Drop）均为现代浏览器标准能力 |
| **离线可用** | 完全不需要网络 |
| **维护成本** | 单文件架构，核心逻辑约 300-500 行 JS，易于修改和扩展 |

## 五、标签提取逻辑

### 5.1 提取流程

```
读取 .js 文件
  ↓
定位 /*: ... */ 注释块
  ↓
从注释块中识别 @help 区域
  ↓
逐行扫描 help 文本
  ↓
匹配行内形如 <TagName ...> 的模式
  ↓
去重（同一插件内同名标签只保留一个原型）
  ↓
归类：按来源插件分组，附加参数描述（若行内包含中文说明则一并提取）
```

### 5.2 匹配规则

**匹配模式：**

```
/<\s*[A-Za-z\u4e00-\u9fa5][A-Za-z\u4e00-\u9fa5\s\-]*:?\s*[^>]*>/g
```

**匹配示例：**

| 原文 | 提取结果 |
|---|---|
| `<Price: x>` | `<Price: x>` |
| `<价格: x>` | `<价格: x>` |
| `<Color: #ffffff>` | `<Color: #ffffff>` |
| `<DM_ACH>` | `<DM_ACH>` |
| `<param Plus: +x>` | `<param Plus: +x>` |

**排除规则：**

- 排除 HTML 标签（如 `<b>`、`<i>`、`<br>`）—— 这些是 RMMZ 的富文本控制符而非备注标签
- 排除 `@type struct<...>` 行中的 `<` —— 这些是插件参数的结构体类型声明，不是备注标签
- 排除 `@` 开头的元数据行

### 5.3 占位符语义保留

提取时保留标签中原有的占位符（`x`、`#ffffff`、`+x%` 等）。插入输入框后，占位符部分被自动选中（高亮），用户可直接键入替换，无需手动选中。

## 六、UI 布局

```
┌──────────────────────────────────────────────────────────┐
│  [导入插件]          已加载: 5 个插件  │  标签总数: 47   │
├────────────────┬─────────────────────────────────────────┤
│  标签面板       │  备注编辑区                            │
│                │                                         │
│  [🔍 搜索标签] │  ┌───────────────────────────────────┐  │
│  [+ 手动添加]  │  │ 标签页 1  [×]                     │  │
│                │  │ ───────────────────────────────── │  │
│  ▼ GF核心      │  │ <Price: 500>                      │  │
│    <Price: x>  │  │ <Color: 3>                        │  │
│    <Color: x>  │  │ <DM_ACH>                          │  │
│    <param: +x> │  │                                   │  │
│  ▼ 成就系统    │  │                                   │  │
│    <DM_ACH>    │  └───────────────────────────────────┘  │
│  ...           │  [+ 标签页]   [复制]  [清空]            │
│  [📦 模板片段] │                                         │
│    <标准武器>  │                                         │
├────────────────┴─────────────────────────────────────────┤
│  [导出标签配置]  [导入标签配置]          Ctrl+S 复制     │
│                                      Ctrl+Shift+S 导出  │
└──────────────────────────────────────────────────────────┘
```

### 6.1 布局要点

1. **顶部状态栏**：导入按钮 + 当前已加载的插件数量 / 标签总数
2. **左侧标签面板**：
   - 搜索框：实时过滤标签
   - 「+ 手动添加」按钮：位于搜索框下方，标签列表上方
   - 标签按来源插件分组，支持折叠/展开
   - 每组显示插件名称、标签数量
   - 每个标签是一个可点击的按钮样式（pill 形状），点击即插入
   - 常用标签自动置顶（按点击频率排序），在分组内排在前面
   - 高亮标签中的占位符值（如 `x`、`#ffffff`）
   - 分组最后一行显示本组的「📦 模板片段」预设
3. **右侧编辑区**：
   - 多标签页：通过「+ 标签页」按钮新增，每个标签页独立命名、独立内容
   - 标签页可关闭（显示 `[×]`）
   - 文本编辑区域支持光标定位
   - 点击标签时在光标位置插入，并自动选中占位符部分
   - 底部按钮：复制、清空
4. **底部操作栏**：导出配置 / 导入配置

### 6.2 快捷键

| 快捷键 | 功能 |
|---|---|
| `Ctrl+F` | 聚焦到标签搜索框 |
| `Ctrl+S` | 复制当前标签页的备注文本到剪贴板 |
| `Ctrl+Shift+S` | 导出当前备注标签配置为 JSON |

### 6.3 导入策略弹窗

```
┌─────────────── 提示 ───────────────┐
│  已有标签配置，如何处理新导入的标签？  │
│                                      │
│  [○] 覆盖 - 仅保留新导入的标签        │
│  [●] 追加 - 保留现有标签，合并新标签   │
│  [  取消  ]                          │
└──────────────────────────────────────┘
```

默认选中「追加」，Enter 确认，ESC 取消。

## 七、扩展功能

### ✅ 高优先级扩展

| 功能 | 实现方式 |
|---|---|
| **标签分组** | 按来源插件名分组，组名可折叠/展开，每组显示标签计数 |
| **常用标签置顶** | 用 `localStorage` 记录每个标签的点击次数，同组内按频率降序排列 |
| **搜索 / 过滤标签** | 实时搜索框，匹配标签名和插件名，支持拼音首字母匹配 |
| **插件信息概览** | 顶栏显示已导入插件列表（鼠标悬停或点击展开详情），各插件标签数量 |
| **按插件过滤** | 在标签搜索框旁提供下拉筛选，指定只看某个插件的标签 |
| **占位符高亮** | 标签中的 `x`、`#ffffff` 等占位符在插入后自动选中，方便替换 |
| **多语言别名配对** | GF 系列大量使用中英双语标签（`<Price: x>` / `<价格: x>`），自动检测并配对展示，点击任一语言版本均插入标签原型 |

### ✅ 中优先级扩展

| 功能 | 实现方式 |
|---|---|
| **多标签页** | 通过「+ 标签页」按钮新增标签页，每个标签页有独立名称（可重命名）和独立的编辑内容；关闭标签页时确认提示 |
| **模板片段** | 在各插件标签分组的最后一行显示预设模板片段（如「标准武器备注」），点击后自动填入一组常用标签的占位符组合 |

### ✅ 低优先级扩展

| 功能 | 实现方式 |
|---|---|
| **快捷键体系** | `Ctrl+F` 搜索、`Ctrl+S` 复制、`Ctrl+Shift+S` 导出，通过 `keydown` 事件绑定 |
| **多语言别名配对** | 自动配对同义中英标签（`<Price: x>` ↔ `<价格: x>`），也支持用户手动建立别名关联 |

### ❌ 已排除的功能

| 功能 | 排除理由 |
|---|---|
| 拖拽排序 | 操作复杂度与收益不成正比 |
| 语法校验 | RMMZ 本身不校验备注格式，强行校验可能误报 |
| 暗色主题 | 当前版本暂不纳入（可在后续版本按需补充 CSS 变量方案） |
| 在线插件库 | 纯离线工具，不做网络请求 |
| 备注预览 | 超出当前工具边界，需引入渲染引擎 |

## 九、开发原则

1. **单文件**：HTML/CSS/JS 各自独立但无需构建工具，浏览器直接运行
2. **渐进增强**：核心功能在 Phase 1 即可用，后续每阶段在上一阶段基础上增量开发
3. **数据在本地**：不产生任何网络请求，用户配置通过文件系统导入导出
4. **UI 优先中文**：界面语言使用中文，与插件内的中文标注保持一致
